# TENSION: Ideology & Manifesto

> "Tension is the space between what you know and what you discover."

---

## 1. Core Philosophy

### Проблема традиционных чатов

Традиционные чат-интерфейсы (ChatGPT, Claude) создают **"туннельное мышление"**:
- Линейная история сообщений
- Если пошли не туда — нужно удалять или начинать заново
- Невозможно сравнить альтернативные ответы
- Контекст теряется при длинных диалогах

### Решение Tension

**Tension** — это **пространственный интерфейс** для работы с LLM:
- **Граф вместо списка** — видите все ветки одновременно
- **Ветвление** — исследуйте альтернативы параллельно
- **Пространственная память** — запоминаете где что находится
- **Бесконечный контекст** — холст не имеет границ

**Мы не строим "диалог", мы строим "карту мыслей".**

## 2. The Enterprise Standard

### 2.1 No Friction (Отсутствие трения)

Интерфейс должен **исчезать**. Пользователь работает с мыслями, а не с UI.

| Метрика | Цель |
|---------|------|
| Время отклика UI | < 16ms (60fps) |
| Время до первого действия | < 200ms |
| Keyboard shortcuts | Для всех частых действий |
| Mouse clicks | Минимум для любой операции |

**Реализация:**
- Оптимистичные обновления (UI обновляется до ответа сервера)
- React.memo для предотвращения лишних рендеров
- CSS transitions вместо JS анимаций
- Debounced сохранение

### 2.2 Infinite Context (Бесконечный контекст)

Мы **не ограничиваем** пользователя.

- Холст бесконечен в любом направлении
- Количество нод не ограничено
- Глубина ветвления не ограничена
- Zoom от 25% до 200%

**Реализация:**
- Виртуализация (рендерим только видимые ноды)
- Lazy loading контента
- Эффективные структуры данных (Map вместо Array)
- Minimap для навигации

### 2.3 Data Integrity (Целостность данных)

Каждая буква, каждая связь сохраняются **мгновенно**.

- Auto-save при каждом изменении
- Undo/Redo для всех операций
- Export в открытом формате (JSON)
- Нет потери данных при крашах

**Реализация:**
- IndexedDB для локального хранения
- History stack для undo/redo
- Transactional updates
- Error boundaries

## 3. User Experience Pillars

### 3.1 Spatial Context

Человек запоминает информацию **пространственно**:
- "Это было справа вверху"
- "Рядом с той красной нодой"
- "В левой ветке"

Мы используем это:
- Ноды имеют фиксированные позиции
- Пользователь сам организует пространство
- Minimap показывает общую картину
- Zoom сохраняет контекст

### 3.2 Augmented Creativity

Мы **не генерируем** текст за пользователя.
Мы **разгоняем** его мышление:

- Ветвление предлагает альтернативы
- Deep Level углубляет исследование
- Визуализация показывает связи
- История сохраняет эксперименты

### 3.3 Professional Grade

Никаких "игрушечных" элементов:

- **Строгий дизайн** — нейтральные цвета, чёткая типографика
- **Токенизированная система** — 100+ CSS переменных
- **Accessibility** — keyboard navigation, ARIA labels
- **Enterprise UI** — Lucide icons, custom components

## 4. Security & Privacy

### 4.1 Local-First Architecture

```
┌─────────────────────────────────────────────────────┐
│                    USER DEVICE                       │
│  ┌─────────────┐    ┌─────────────┐                 │
│  │   Browser   │    │  IndexedDB  │                 │
│  │   (React)   │◄──►│   (Data)    │                 │
│  └──────┬──────┘    └─────────────┘                 │
│         │                                            │
│         │ HTTPS (Direct)                            │
│         ▼                                            │
│  ┌─────────────┐                                    │
│  │  OpenAI API │  ◄── API Key stored locally        │
│  └─────────────┘                                    │
└─────────────────────────────────────────────────────┘
```

**Нет промежуточных серверов.** Данные не покидают устройство пользователя (кроме запросов к AI API).

### 4.2 API Key Security

- Ключ хранится **только** в IndexedDB браузера
- Никогда не отправляется на наши серверы
- Не логируется и не трекается
- Пользователь может удалить в любой момент

### 4.3 Open Source Transparency

- Весь код открыт для аудита
- Нет обфускации или минификации в dev mode
- MIT лицензия
- Нет скрытой телеметрии

## 5. Design Principles

### 5.1 Visual Hierarchy

```
1. Primary Actions    → Яркие, заметные (Play button)
2. Secondary Actions  → Нейтральные (Toolbar buttons)
3. Tertiary Actions   → Скрытые (Context menu)
4. Content            → Максимальный контраст для читаемости
5. Chrome             → Минимальный визуальный вес
```

| Primary | #6366f1 | #6366f1 |

## 6. Technical Excellence

### 6.1 Architecture (FSD)

Feature-Sliced Design обеспечивает:
- **Изоляцию** — каждый слой независим
- **Масштабируемость** — легко добавлять фичи
- **Тестируемость** — чёткие границы
- **Понятность** — новый разработчик быстро разберётся

### 6.2 Type Safety

```typescript
// Строгая типизация всех сущностей
interface Node {
  id: string;
  x: number;
  y: number;
  prompt: string;
  // ...
}

// Нет any, нет unknown без проверки
// Все события типизированы
// Все props обязательны или имеют defaults
```
